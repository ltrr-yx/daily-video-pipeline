# Production Principles

## Audience Output Boundary

Internal editorial guidance is never audience copy.

Do not render or narrate internal labels such as `story.template`, `component`, `visual_grammar`, `motion_grammar`, prompt notes, plate roles, layout contracts, renderer phases, review instructions, or user-provided editorial guidance. These terms can appear in config, manifest, docs, gallery, CLI output, and review tooling, but not inside the final video frame, voiceover, captions, script lines intended for the audience, or publishing copy.

When a user gives visual direction, editorial guidance, prompt constraints, or implementation notes, treat those as production instructions. Convert them into viewer-facing claims only when the user explicitly asks for audience copy and the wording is safe, factual, and relevant to the viewer.

Final-frame text and narration should use audience-facing language: the event, evidence, metric, consequence, source, and next watch item. Keep authoring vocabulary out of the finished piece.

Audience-facing UI labels, badges, pills, headers, and lower-thirds must describe reader-visible context such as project name, date, source, topic, evidence, or recap purpose. They must not show internal story, scene, visual, motion, component, template, or grammar choices.
